# Отчитане на праховото боядисване

Самостоятелен инструмент: операторът въвежда какво има на линията, а системата
показва **какво е боядисано и какво остава** на затворения конвейр в реално време.

Един файл, без библиотеки — `index.html` се отваря с двоен клик.

## Какво прави

- **Видове подвески** — двойна/тройна/четворна/петторна/висяща, с редактируем
  брой редове (напречни железа) и цвят.
- **Детайли (бутони)** — каталог детайли с „бр/ред"; маркираш ред от състава и
  натискаш детайл, за да го заредиш.
- **Състав на линията сега** — редове „тип подвеска + детайл + брой"; сменя се по
  всяко време през деня (по-малко тройни, повече за крака и т.н.).
- **Затворен конвейр на живо** — подвеските се движат по примката и минават през
  неподвижната кабина, където стават „боядисани".
- **Отчитане** — боядисани/остават (общо и по детайл, и в брой детайли), процент,
  изтекло време, паузи и оставащо време (ETA).
- **Боя** — текущ цвят + брой смени за деня (за себестойността).

## Управление

- **Старт / Пауза** — пуска движението; Пауза = почивка (за реалното време).
- **Нулирай** — нова партида.
- **Скорост** (подв/час) и **Преглед ×** (само за демонстрация — върти по-бързо).
- **Корекция** (мина + / −) — сверява брояча с реалността при засечка.

## Модел на данните (за разработка)

```js
hangerTypes = [{ id, name, rows, color }]   // видове подвески
parts       = [{ id, name, perRow }]        // детайли (бр/ред)
entries     = [{ hanger, part, count }]      // състав на линията сега
hangers     = []   // строи се от entries → въртят се по контура
// капацитет на 1 подвеска = hanger.rows * part.perRow
// боядисани = подвески, минали кабината (phi); остават = другите
```

## Идеи за разширение

- **Смяна на цвят = нова партида** с лог на смените за деня.
- **История на партидите** (запис какво е боядисано, кога, колко).
- Връзка към **себестойност**: детайли/час × (ставка боядисване + прах/брой).
- **Брояч/сензор на кабината** (или баркод при подаване) → автоматично „мина +"
  вместо ръчно.

## Точност — честно

Чисто по часовник броячът може да се размине при дребни спирания. Затова са важни
**Пауза** и ръчната корекция. За реални (не оценъчни) данни — сензор/брояч на
кабината.
