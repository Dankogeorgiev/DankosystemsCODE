# Данко Системс — Следене на мостри

Уеб-приложение за следене на изпълнението на мострите в Данко Системс.
**Споделена версия**: няколко души влизат с парола и виждат/редактират едни
и същи мостри. Данните и чертежите се пазят в облака (Supabase), а интерфейсът
работи директно в браузъра.

## Как се пуска

1. Настрой облака веднъж по **[`SETUP.md`](SETUP.md)** (Supabase — безплатно)
   и попълни `config.js` с твоите данни за връзка.
2. Отвори приложението (онлайн линка от GitHub Pages или `index.html` локално).
3. Влез с имейл и парола → готово.

> Без попълнен `config.js` приложението ще покаже съобщение, че липсва настройка.

## Какво съдържа всяка мостра

1. **Име на клиента**
2. **Информация за клиента** — свободно поле
3. **Информация за мострата** — свободно поле
4. **Чертежи за производство** — прикачване на изображения и PDF
5. **Необходими материали** — таблица (материал, количество, забележка)
6. **Поръчка на материали** — статус, доставчик, забележки
7. **Производствен процес** — тикче + отговорник за всяка операция:
   - Лазерно рязане
   - Огъване
   - Заварване
   - Допълнителни операции
   - Боядисване
   - Сглобяване
8. **Анализ на производството** — какво е направило впечатление

## Функции

- **Вход с парола** и **споделени данни** между всички потребители
- Промените се появяват при останалите **на живо** (realtime)
- Споделени **чертежи** (снимки/PDF) в облачно хранилище
- Списък с всички мостри + търсене
- **Краен срок** на мострата и **дата на поръчката** / очаквана доставка
- **Статус „Завършена“** + значка в списъка
- **Обща справка** за всички мостри (общ брой, в процес, завършени, просрочени) с печат/PDF
- Индикатор за напредък по операциите
- Автоматично запазване при всяка промяна
- Експорт на всички данни (JSON) за резервно копие; импорт от JSON
- Печат / запис като PDF на отделна мостра

## Важно за данните

Данните се пазят в облака (Supabase). Достъп имат само хора с акаунт, който
ти си създал. Безплатният план на Supabase е достатъчен за малък екип.
